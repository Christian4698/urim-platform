# PLANS.md

Toute tâche dépassant un changement local doit créer un plan dans `docs/exec-plans/active/`.

## Modèle
- Objectif
- Portée / hors portée
- Hypothèses
- Sources de données concernées
- Risques de fuite temporelle
- Schémas ou migrations
- Étapes
- Tests
- Observabilité
- Sécurité
- Plan de retour arrière
- Critères d’acceptation
- Décisions prises
- État d’avancement

Une fois terminé, déplacer le plan dans `docs/exec-plans/completed/`.
