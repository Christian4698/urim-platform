# Fiabilité et observabilité

## SLI
Disponibilité, latence p50/p95/p99, fraîcheur, erreurs, quotas, conflits, taux NO_BET, temps de calcul, drift et calibration rolling.

## Trace
`provider request → observation → normalisation → feature snapshot → modèle → risque → publication`.

## Alertes
Provider down, données stales, divergence, quota faible, schéma changé, latence live, drift, calibration dégradée, hausse anormale des conseils ou ledger non vérifiable.

Une source critique indisponible entraîne `INSUFFICIENT_DATA`.
