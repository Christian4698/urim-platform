# backtesting-audit


1. Lire `docs/11_BACKTESTING_PROTOCOL.md`.
2. Rejouer seulement les données disponibles à l’époque.
3. Simuler latence, indisponibilité, marge et contraintes.
4. Journaliser tous les essais.
5. Réserver un test final vierge.
6. Produire intervalles et résultats segmentés.
7. Lister les risques de sélection opportuniste.


## Sortie attendue
- Résumé
- Fichiers modifiés
- Tests exécutés
- Risques restants
- IDs E001–E084 concernés
